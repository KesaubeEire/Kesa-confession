def Login_in_with_URL(url = "http://nladuo.cn/crawler_lesson2/do_login.php",
                      data = {"uname": "nladuo", "passwd": "nladuo"}):
# 1. 先登陆
resp1 = requests.post(url, data)

# 2. 保存服务器传回来的Cookie

print(resp1.text)

print("Set-Cookie:", resp1.headers["set-cookie"])
cookie = resp1.headers["set-cookie"].split(";")[0]

# 3. 再通过cookie请求隐私页面
# 现用浏览器或者Telnet发送Post请求登录, 把cookie粘到这里
resp = requests.get("http://nladuo.cn/crawler_lesson2/private.php", 
                    headers = {"Cookie": cookie})
soup = BeautifulSoup(resp.content, features = "lxml")
print(soup)

爬虫框架的需求:

- 任务产生器——Producer  
    - 定义任务，如：爬取什么页面？怎么解析
- 下载器——Downloader  
    - 接受任务产生器的任务，下载完成后给解析器进行解析。主要是I/O操作，受限于网速。
- 解析器——Parser  
    - 将下载器下载的内容进行解析，传给输出管道。主要是CPU操作，受限于下载器的下载速度。
- 输出管道——Pipeline  
    - 如何展示爬取的数据

CPU_密集型 和 IO_密集型业务:

系统整体的运行速度其实受限于速度最慢的那个。
像我们爬虫，最耗时的操作就是下载，整体的爬取速度也就受限于网速。
多线程可以实现多重下载,可以加速整个爬虫最慢的过程.# 这句是我自己想的

以上的生产和消费者类似爬虫中的 Producer 和 Worker 。
Producer 扮演生产者，生成下载任务，放入任务队列中；
Worker   扮演消费者，拿到下载任务后，对某个网页进行下载、解析、数据；
在此同时，Worker 也会扮演生产者，根据解析到的链接生成新的下载任务，并放到任务队列中交给其他的 Worker 执行。
